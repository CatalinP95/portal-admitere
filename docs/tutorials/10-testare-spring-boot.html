2. @SpringBootTest – contextul complet

@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.MOCK)
// MOCK = simulează HTTP fără a porni serverul pe un port real
// RANDOM_PORT = pornește serverul pe un port aleator (testare reală HTTP)
// DEFINED_PORT = pornește pe portul din application.yml
// NONE = fără server (testare servicii pure)
@AutoConfigureMockMvc    // injectează MockMvc automat
@ActiveProfiles("test")  // încarcă application-test.yml (H2 în loc de MySQL)
class UserProfileControllerTest {

    @Autowired private MockMvc mockMvc;
    @Autowired private JwtUtil jwtUtil;
    @Autowired private UserRepository userRepository;
    @Autowired private UserProfileRepository userProfileRepository;
    @Autowired private PasswordEncoder passwordEncoder;
    @Autowired private ObjectMapper objectMapper;
    @MockBean private AuditLogService auditLogService;
    // @MockBean înlocuiește AuditLogService cu un mock Mockito în contextul Spring
}

3. H2 – baza de date in-memory pentru teste

H2 este o bază de date Java care rulează în memorie — nu necesită instalare, pornește instant, se resetează între teste.

# application-test.yml
spring:
  datasource:
    url: jdbc:h2:mem:testdb;DB_CLOSE_DELAY=-1;DB_CLOSE_ON_EXIT=FALSE
    # mem:testdb = in-memory, DB_CLOSE_DELAY=-1 = nu se închide
    driver-class-name: org.h2.Driver
    username: sa
    password:                   # parola goală pentru H2
  jpa:
    hibernate:
      ddl-auto: create          # recreează schema înainte de teste
    database-platform: org.hibernate.dialect.H2Dialect
  autoconfigure:
    exclude:
      - org.springframework.boot.autoconfigure.amqp.RabbitAutoConfiguration
      - org.springframework.boot.autoconfigure.mongo.MongoAutoConfiguration
      - org.springframework.boot.autoconfigure.data.mongo.MongoDataAutoConfiguration
      - org.springframework.boot.autoconfigure.data.mongo.MongoRepositoriesAutoConfiguration
      - org.springframework.boot.autoconfigure.data.redis.RedisAutoConfiguration
      - org.springframework.boot.autoconfigure.data.redis.RedisRepositoriesAutoConfiguration
  cache:
    type: none                  # dezactivăm Redis cache

De ce excludem MongoDB, Redis, RabbitMQ în teste?
Aceste servicii externe nu rulează în mediul de CI/test. Dacă le lăsăm active, Spring încearcă să se conecteze și testul eșuează. Le excludem din autoconfigurare și mock-uim serviciile care le folosesc (ex: AuditLogService).

4. @BeforeEach – pregătirea datelor de test

private String studentToken;
private String adminToken;
private Long studentId;
private Long adminId;

@BeforeEach
void setUp() {
    // IMPORTANT: ordinea de ștergere respectă foreign keys!
    // Mai întâi ștergem entitățile cu FK
    userProfileRepository.deleteAll();
    refreshTokenRepository.deleteAll();
    // Abia apoi ștergem userul (referenced de celelalte)
    userRepository.deleteAll();

    // Creăm un user student
    User student = new User();
    student.setUsername("student");
    student.setEmail("student@test.com");
    student.setPassword(passwordEncoder.encode("student123"));
    // Parola trebuie hash-uită — Spring Security compară hash-uri, nu text plain
    student.setRole(Role.STUDENT);
    student.setEnabled(true);
    studentId = userRepository.save(student).getId();

    // Generăm JWT pentru test (fără login real)
    studentToken = jwtUtil.generateToken(String.valueOf(studentId), "STUDENT");

    User admin = new User();
    admin.setUsername("admin");
    admin.setEmail("admin@test.com");
    admin.setPassword(passwordEncoder.encode("admin123"));
    admin.setRole(Role.ADMIN);
    admin.setEnabled(true);
    adminId = userRepository.save(admin).getId();
    adminToken = jwtUtil.generateToken(String.valueOf(adminId), "ADMIN");
}

5. MockMvc – testarea REST endpoint-urilor

5.1 Structura unui test MockMvc

@Test
void saveProfile_asStudent_returns200() throws Exception {
    // Pregătim request body
    UserProfileRequest req = new UserProfileRequest();
    req.setFirstName("Ion");
    req.setLastName("Popescu");
    req.setCnp("1990101123456");
    req.setPhone("+40722123456");

    mockMvc.perform(
            put("/api/profile")                                     // metoda HTTP + URL
                .header("Authorization", "Bearer " + studentToken)    // JWT token
                .contentType(MediaType.APPLICATION_JSON)               // Content-Type header
                .content(objectMapper.writeValueAsString(req))          // body serializat JSON
        )
        .andExpect(status().isOk())                                      // HTTP 200
        .andExpect(jsonPath("$.firstName").value("Ion"))              // câmp în JSON response
        .andExpect(jsonPath("$.lastName").value("Popescu"))
        .andExpect(jsonPath("$.cnp").value("1990101123456"));
}

6. @MockBean vs @Mock

@Mock (Mockito)@MockBean (Spring Boot Test)
ContextFără Spring, test pur MockitoÎnlocuiește bean-ul în contextul Spring
Unde se folosește@ExtendWith(MockitoExtension.class)@SpringBootTest
Injecție@InjectMocks injectează în obiectSpring injectează automat
ScopUnit tests izolateIntegration tests cu Spring context
// Unit test cu @Mock — fără Spring
@ExtendWith(MockitoExtension.class)
class AuthServiceTest {
    @Mock private UserRepository userRepository;
    @Mock private JwtUtil jwtUtil;
    @InjectMocks private AuthService authService;
    // @InjectMocks creează AuthService și injectează mock-urile
}

// Integration test cu @MockBean — cu Spring context
@SpringBootTest
@ActiveProfiles("test")
class ProfileControllerTest {
    @MockBean private AuditLogService auditLogService;
    // Spring înlocuiește AuditLogService real cu un mock în tot contextul
}

7. Mockito – cum mock-uim metode

// when().thenReturn() — stub pentru metode care returnează ceva
when(userRepository.findByUsername("ion"))
    .thenReturn(Optional.of(user));

when(jwtUtil.generateToken("1", "STUDENT"))
    .thenReturn("mock-token-123");

// when().thenThrow() — stub pentru excepții
when(authenticationManager.authenticate(any()))
    .thenThrow(new BadCredentialsException("Bad credentials"));

// doNothing().when() — stub pentru metode void
doNothing().when(refreshTokenRepository).revokeAllByUser(any());

// verify() — verificăm că o metodă a fost apelată
verify(userRepository, times(1)).save(any(User.class));
verify(userRepository, never()).delete(any());

// ArgumentMatchers — matchers pentru argumente
when(userRepository.existsByUsername(anyString())).thenReturn(false);
when(repo.findById(eq(1L))).thenReturn(Optional.of(user));

8. Unit Test complet – AuthServiceTest

@ExtendWith(MockitoExtension.class)
class AuthServiceTest {

    @Mock private UserRepository userRepository;
    @Mock private RefreshTokenRepository refreshTokenRepository;
    @Mock private PasswordEncoder passwordEncoder;
    @Mock private JwtUtil jwtUtil;
    @Mock private AuthenticationManager authenticationManager;

    @InjectMocks
    private AuthService authService;

    @Test
    void register_success() {
        RegisterRequest req = new RegisterRequest();
        req.setUsername("ion");
        req.setEmail("ion@test.com");
        req.setPassword("pass123");

        // Stub-uri
        when(userRepository.existsByUsername("ion")).thenReturn(false);
        when(userRepository.existsByEmail("ion@test.com")).thenReturn(false);
        when(passwordEncoder.encode("pass123")).thenReturn("hash123");

        User saved = new User();
        saved.setId(1L);
        saved.setRole(Role.STUDENT);
        when(userRepository.save(any())).thenReturn(saved);
        when(jwtUtil.generateToken("1", "STUDENT")).thenReturn("access-token");
        when(jwtUtil.generateRefreshToken("1", "STUDENT")).thenReturn("refresh-token");
        when(refreshTokenRepository.save(any())).thenReturn(new RefreshToken());

        // Execuție
        AuthResponse response = authService.register(req);

        // Verificări
        assertNotNull(response);
        assertEquals("access-token", response.getAccessToken());
        assertEquals("STUDENT", response.getRole());
        verify(userRepository, times(1)).save(any());
    }

    @Test
    void register_duplicateUsername_throws() {
        RegisterRequest req = new RegisterRequest();
        req.setUsername("existing");

        when(userRepository.existsByUsername("existing")).thenReturn(true);

        assertThrows(IllegalArgumentException.class, () -> authService.register(req));
        verify(userRepository, never()).save(any());
        // verify că save NU a fost apelat
    }
}

9. ReflectionTestUtils – testarea claselor cu @Value

Când o clasă folosește @Value și nu vrem să pornim Spring complet, putem seta câmpurile direct prin reflexie.

// JwtUtilTest — fără Spring context, fără @RefreshScope issues
class JwtUtilTest {
    private JwtUtil jwtUtil;

    @BeforeEach
    void setUp() {
        jwtUtil = new JwtUtil();  // instanțăm direct, fără Spring
        // Setăm câmpurile private cu ReflectionTestUtils
        ReflectionTestUtils.setField(jwtUtil, "secret",
            "404E635266556A586E3272357538782F413F4428472B4B6250645367566B5970");
        ReflectionTestUtils.setField(jwtUtil, "expiration", 86400000L);
        ReflectionTestUtils.setField(jwtUtil, "refreshExpiration", 604800000L);
    }

    @Test
    void generateToken_isValid() {
        String token = jwtUtil.generateToken("42", "STUDENT");
        assertTrue(jwtUtil.isTokenValid(token));
    }
}

10. Tipare comune de teste

Test pentru autorizare (403)

@Test
void getProfile_noToken_returns403() throws Exception {
    mockMvc.perform(get("/api/profile"))  // fără header Authorization
            .andExpect(status().isForbidden());
}

@Test
void getAllUsers_asStudent_returns403() throws Exception {
    mockMvc.perform(get("/api/users")
                    .header("Authorization", "Bearer " + studentToken))
            .andExpect(status().isForbidden());
            // student nu are acces la /api/users (doar ADMIN)
}

Test pentru validare (400)

@Test
void saveProfile_invalidCnp_returns400() throws Exception {
    UserProfileRequest req = new UserProfileRequest();
    req.setCnp("123");  // CNP invalid (nu are 13 cifre)

    mockMvc.perform(put("/api/profile")
                    .header("Authorization", "Bearer " + studentToken)
                    .contentType(MediaType.APPLICATION_JSON)
                    .content(objectMapper.writeValueAsString(req)))
            .andExpect(status().isBadRequest());
            // @Valid pe controller returnează 400 pentru CNP invalid
}

11. Întrebări posibile la examen

Q: Care este diferența dintre @Mock și @MockBean?
@Mock (Mockito pură) creează un mock fără Spring — se folosește în unit tests cu @ExtendWith(MockitoExtension.class). @InjectMocks injectează mock-urile în clasa testată. @MockBean (Spring Boot Test) înlocuiește bean-ul real din contextul Spring cu un mock Mockito — se folosește în @SpringBootTest. Dacă un bean depinde de un serviciu extern (MongoDB, Redis), @MockBean pe acel serviciu previne erorile de context.
Q: De ce folosim H2 în loc de MySQL pentru teste?
H2 este o bază de date in-memory (rulează în JVM, fără instalare). Avantaje: pornește instant, se izolează între teste (ddl-auto: create recreează schema), nu necesită conexiune la un server extern, funcționează în CI/CD fără setup. Dezavantaj: dialectul diferă de MySQL (unele funcții SQL specifice MySQL nu funcționează în H2). Pentru teste care necesită funcții MySQL specifice, poți folosi Testcontainers (MySQL real în Docker).
Q: Ce face @BeforeEach și de ce e importantă ordinea de ștergere?
@BeforeEach rulează înainte de fiecare test @Test, asigurând o stare curată a bazei de date. Ordinea de ștergere contează din cauza foreign key constraints: dacă UserProfile are user_id FK care referențiază Users, nu poți șterge un User dacă există un UserProfile asociat. Soluție: ștergi mai întâi entitățile cu FK (userProfileRepository.deleteAll(), refreshTokenRepository.deleteAll()) și abia apoi userul.
Q: Ce este MockMvc și cum funcționează?
MockMvc simulează requesturi HTTP fără a porni serverul real. Testele trimit requesturi fake și verifică răspunsurile. Avantaj față de un server real: mai rapid, context izolat, nu ocupă port. @AutoConfigureMockMvc configurează MockMvc automat cu toată configurarea de Security. Testele cu MockMvc sunt integration tests reale (folosesc contextul Spring complet, baza de date H2 reală).
Q: De ce excludem MongoAutoConfiguration și RedisAutoConfiguration în application-test.yml?
MongoDB și Redis sunt servicii externe care nu rulează în mediul de test. Dacă autoconfigurarea este activă, Spring încearcă să se conecteze la ele și aruncă excepție dacă nu găsește conexiune. Excluzând autoconfigurarea, beans-urile MongoTemplate și RedisTemplate nu sunt create. Dar AuditLogRepository (MongoRepository) are nevoie de MongoTemplate — de aceea adăugăm și MongoRepositoriesAutoConfiguration în excluderi, și mock-uim AuditLogService cu @MockBean.
Q: Ce este ReflectionTestUtils.setField și când îl folosești?
ReflectionTestUtils este o utilitate Spring Test care permite setarea câmpurilor private prin reflexie, fără Spring context. Se folosește când vrei să testezi o clasă care are @Value dar nu vrei să pornești Spring complet (ex: JwtUtil cu @RefreshScope — Spring Cloud scope-ul nu e disponibil în contextul minimal). Alternativa este field injection cu @Autowired sau constructor injection care permite testare directă.

12. Implementare pas cu pas — Testare Spring Boot

Prerequisite: spring-boot-starter-test în pom.xml (inclus implicit). spring-security-test pentru teste cu autentificare.

Pasul 1 — Configurează application-test.yml

Creează src/test/resources/application-test.yml — folosește H2 în loc de MySQL:

spring:
  datasource:
    url: jdbc:h2:mem:testdb;DB_CLOSE_DELAY=-1
    driver-class-name: org.h2.Driver
    username: sa
    password:
  jpa:
    database-platform: org.hibernate.dialect.H2Dialect
    hibernate:
      ddl-auto: create-drop   # recreează schema la fiecare rulare de teste
  autoconfigure:
    exclude:
      - org.springframework.boot.autoconfigure.amqp.RabbitAutoConfiguration
      - org.springframework.boot.autoconfigure.mongo.MongoAutoConfiguration
      - org.springframework.boot.autoconfigure.data.mongo.MongoDataAutoConfiguration
      - org.springframework.boot.autoconfigure.data.mongo.MongoRepositoriesAutoConfiguration
      - org.springframework.boot.autoconfigure.data.redis.RedisAutoConfiguration
  cache:
    type: none   # dezactivează Redis cache în teste

jwt:
  secret: 404E635266556A586E3272357538782F413F4428472B4B6250645367566B5970
  expiration: 86400000
  refresh-expiration: 604800000

Excluderile sunt necesare deoarece H2 nu suportă RabbitMQ/MongoDB/Redis — Spring încearcă să se conecteze la startup și testele eșuează fără ele.

Pasul 2 — Adaugă dependința spring-security-test 
<dependency>

Pasul 3 — Creează clasa de test de bază

Creează src/test/java/com/campus/userservice/UserControllerTest.java:

@SpringBootTest
@AutoConfigureMockMvc
@ActiveProfiles("test")   // folosește application-test.yml
class UserControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @Autowired
    private UserRepository userRepository;

    @Autowired
    private JwtUtil jwtUtil;

    private String adminToken;

    @BeforeEach
    void setUp() {
        userRepository.deleteAll();   // curăță DB înainte de fiecare test

        // Creează un user ADMIN și generează token JWT pentru el
        User admin = new User();
        admin.setUsername("admin");
        admin.setPassword(new BCryptPasswordEncoder().encode("pass"));
        admin.setEmail("admin@test.com");
        admin.setRole(Role.ADMIN);
        admin = userRepository.save(admin);

        adminToken = jwtUtil.generateToken(admin.getId().toString(), "ADMIN");
    }

    @Test
    void getUsers_asAdmin_returns200() throws Exception {
        mockMvc.perform(get("/api/users")
                .header("Authorization", "Bearer " + adminToken))
                .andExpect(status().isOk())
                .andExpect(jsonPath("$.content").isArray());
    }

    @Test
    void getUsers_withoutToken_returns401() throws Exception {
        mockMvc.perform(get("/api/users"))
                .andExpect(status().isUnauthorized());
    }
}
Pasul 4 — Rulează testele

Din VS Code: click dreapta pe clasa de test → Run Tests. Sau din terminal:
