JwtInterceptor – Functional Interceptor

Angular 17+ folosește functional interceptors în loc de clase. Se înregistrează în app.config.ts prin withInterceptors([jwtInterceptor]).

export const jwtInterceptor: HttpInterceptorFn = (req, next) => {
  const auth   = inject(AuthService);
  const router = inject(Router);
  const token  = auth.getToken();

  const authReq = token
    ? req.clone({ setHeaders: { Authorization: `Bearer ${token}` } })
    : req;

  return next(authReq).pipe(
    catchError((err: HttpErrorResponse) => {
      const isAuthEndpoint = req.url.includes('/auth/');
      if (err.status === 401 && !isAuthEndpoint) {
        auth.clearTokens();        // curăță local, fără request suplimentar
        router.navigate(['/login']);
      }
      return throwError(() => err);
    })
  );
};

Regulă critică — excluderea rutelor /auth/: Interceptorul NU trebuie să trateze 401-ul de pe /auth/login ca sesiune expirată. Un 401 la login înseamnă credențiale greșite, nu token invalid. Fără excludere, interceptorul ar apela clearTokens() + redirect la /login pe fiecare parolă greșită, creând o buclă.

Nu apela auth.logout() din interceptor: logout() face un request HTTP către /auth/logout. Dacă token-ul e deja invalid, acel request poate eșua cu 401 sau 500 — creând un lanț de requesturi eșuate. Soluție: curăță tokenurile local cu clearTokens() (sincron, fără request) și redirecționează direct.

AuthGuard – CanActivateFn

Guard-ul verifică dacă există un token valid înainte de a permite accesul la o rută. Dacă nu există token, redirecționează la /login.

export const authGuard: CanActivateFn = () => {
  const auth   = inject(AuthService);
  const router = inject(Router);

  if (auth.isLoggedIn()) return true;

  router.navigate(['/login']);
  return false;
};

RoleGuard – Protejare pe bază de rol

RoleGuard citește rolurile permise din route.data['roles'] și verifică dacă rolul utilizatorului e inclus.

export const roleGuard: CanActivateFn = (route: ActivatedRouteSnapshot) => {
  const requiredRoles: string[] = route.data['roles'] ?? [];
  const userRole = auth.getUserRole();

  if (requiredRoles.length === 0 || requiredRoles.includes(userRole ?? '')) {
    return true;
  }
  router.navigate(['/unauthorized']);
  return false;
};

Lazy loading: Toate componentele folosesc loadComponent cu import dinamic. Angular descarcă codul unei componente doar când ruta e accesată, nu la pornirea aplicației. Reduce dimensiunea bundle-ului inițial.

Diferența dintre AuthGuard și JwtInterceptor

AuthGuardJwtInterceptor
Rulează înainte de a încărca componentaRulează la fiecare request HTTP
Verifică dacă token-ul existăAdaugă token-ul în header
Blochează navigareaGestionează 401 de la server
Protecție la nivel de rutăProtecție la nivel de request
Token expirat dar prezent → permite accesulServer returnează 401 → curăță și redirecționează

De ce avem nevoie de ambele? AuthGuard verifică rapid dacă token-ul există în storage (verificare locală, fără request). Dacă token-ul există dar e expirat, guard-ul îl lasă să treacă — serverul va returna 401, pe care interceptorul îl prinde și face cleanup. Cele două se completează.

Întrebări examen

1. Ce face un HTTP Interceptor în Angular?
Un interceptor interceptează toate requesturile HTTP înainte de a fi trimise și toate răspunsurile înainte de a ajunge la componentă. Poate modifica requestul (adăugă headere), poate transforma răspunsul sau poate trata erorile centralizat. E echivalentul unui middleware HTTP pe frontend.
2. Care este diferența dintre CanActivate și CanLoad?
CanActivate rulează înainte de a activa o rută — componenta e deja descărcată dar nu e afișată. CanLoad previne descărcarea codului lazy-loaded dacă utilizatorul nu e autorizat. În Angular 15+, CanLoad e deprecat în favoarea CanMatch. În proiect folosim CanActivate care e suficient pentru rutele protejate.
3. Ce este lazy loading și ce avantaj aduce?
Lazy loading înseamnă că codul unei componente/modul e descărcat doar când utilizatorul navighează la ruta respectivă, nu la pornirea aplicației. Avantaj: bundle-ul inițial e mai mic → aplicația pornește mai repede. În Angular, se implementează cu loadComponent + import dinamic.
4. De ce interceptorul nu trebuie să apeleze logout() la 401 pe /auth/login?
Un 401 pe /auth/login înseamnă credențiale greșite — utilizatorul a introdus parola greșit. Dacă interceptorul ar apela logout() pentru orice 401, ar curăța tokenurile și ar redirecționa la /login chiar și când utilizatorul tocmai încearcă să se autentifice. Soluție: excludem rutele /auth/ de la tratamentul automat al 401.
5. Cum protejezi o rută pe bază de rol în Angular?
Se folosește CanActivateFn (RoleGuard) care citește rolurile permise din route.data['roles'] și verifică dacă rolul utilizatorului curent e în acea listă. În configurația rutei se adaugă canActivate: [roleGuard] și data: { roles: ['ADMIN'] }.
6. Ce se întâmplă dacă token-ul JWT există în storage dar e expirat?
AuthGuard verifică doar existența token-ului (!!getToken()), nu validitatea lui. Dacă token-ul există dar e expirat, guard-ul permite accesul. Primul request HTTP returnează 401 de la server. Interceptorul prinde 401-ul, curăță token-ul din storage și redirecționează la /login. Cele două straturi se complementează.
7. De ce clearTokens() e mai sigur decât logout() în interceptor?
logout() face un request HTTP către /auth/logout care poate eșua dacă token-ul e deja invalid (server returnează 500 sau 401). Aceasta poate crea un lanț de requesturi eșuate sau o buclă. clearTokens() operează sincron pe localStorage/sessionStorage — nu depinde de server și nu poate eșua.
8. Cum funcționează ruta wildcard ** în Angular Router?
Ruta ** se potrivește cu orice URL care nu a fost prins de rutele anterioare. Angular Router parcurge rutele în ordine — primul match câștigă. De aceea ** trebuie să fie ultimă în array. Dacă e prima, toate navigările ar duce la pagina 404, indiferent de URL.

Implementare pas cu pas — Interceptor, Guards și Dashboard

Prerequisite: AuthService creat (Tutorial 13). Rutele de bază configurate în app.routes.ts.

Pasul 1 — Creează JWT Interceptor 
ng generate interceptor interceptors/jwt

Editează src/app/interceptors/jwt.interceptor.ts:

import { HttpInterceptorFn } from '@angular/common/http';
import { inject } from '@angular/core';
import { AuthService } from '../services/auth.service';
import { Router } from '@angular/router';
import { catchError, throwError } from 'rxjs';

export const jwtInterceptor: HttpInterceptorFn = (req, next) => {
  const authService = inject(AuthService);
  const router = inject(Router);

  const token = authService.getToken();

  // Adaugă token-ul la fiecare request (dacă există)
  const authReq = token
    ? req.clone({ setHeaders: { Authorization: `Bearer ${token}` } })
    : req;

  return next(authReq).pipe(
    catchError(err => {
      if (err.status === 401) {
        // Token expirat sau invalid → logout și redirect la login
        authService.logout();
        router.navigate(['/login']);
      }
      return throwError(() => err);
    })
  );
};
Pasul 2 — Creează AuthGuard și RoleGuard 
ng generate guard guards/auth
ng generate guard guards/role

Editează src/app/guards/auth.guard.ts:

import { CanActivateFn } from '@angular/router';
import { inject } from '@angular/core';
import { AuthService } from '../services/auth.service';
import { Router } from '@angular/router';

export const authGuard: CanActivateFn = (route, state) => {
  const authService = inject(AuthService);
  const router = inject(Router);

  if (authService.isLoggedIn()) {
    return true;
  }
  router.navigate(['/login']);
  return false;
};

Editează src/app/guards/role.guard.ts:

export const roleGuard: CanActivateFn = (route, state) => {
  const authService = inject(AuthService);
  const router = inject(Router);

  const allowedRoles: string[] = route.data['roles'] ?? [];
  const userRole = authService.getUserRole();   // extrage rolul din JWT payload

  if (allowedRoles.includes(userRole)) {
    return true;
  }
  router.navigate(['/unauthorized']);
  return false;
};
Pasul 3 — Configurează rutele cu Guards și Lazy Loading

Editează src/app/app.routes.ts:

export const routes: Routes = [
  { path: 'login', loadComponent: () => import('./features/auth/login/login')
      .then(m => m.LoginComponent) },

  { path: 'dashboard', canActivate: [authGuard],
    loadComponent: () => import('./features/dashboard/layout/layout')
      .then(m => m.DashboardLayoutComponent),
    children: [
      { path: 'users', canActivate: [roleGuard], data: { roles: ['ADMIN'] },
        loadComponent: () => import('./features/dashboard/users/users')
          .then(m => m.UsersComponent) },
      { path: 'profile', canActivate: [roleGuard], data: { roles: ['ADMIN', 'STUDENT', 'SECRETARIAT'] },
        loadComponent: () => import('./features/dashboard/profile/profile')
          .then(m => m.ProfileComponent) },
    ]},

  { path: 'unauthorized', loadComponent: () => import('./features/errors/unauthorized/unauthorized')
      .then(m => m.UnauthorizedComponent) },
  { path: '**', loadComponent: () => import('./features/errors/not-found/not-found')
      .then(m => m.NotFoundComponent) }   // ** trebuie să fie ULTIMA rută
];

loadComponent: () => import(...) = lazy loading — componenta se descarcă din rețea doar când utilizatorul navighează la ea, nu la startup.
