📄 Pageable & Sorting

Paginare și sortare cu Spring Data JPA – fără să scrii SQL manual

De ce paginare?

Dacă ai 10.000 de înregistrări și returnezi toate la un request GET, baza de date și rețeaua sunt suprasolicitate. Paginarea returnează un subset (ex: primele 10) cu posibilitatea de a naviga la pagina următoare.

Concret: GET /api/users retornează toate recordurile. GET /api/users?page=0&size=10&sort=lastName,asc returnează primele 10 useri, sortați după nume.

Controller – primim Pageable automat

// UserProfileController.java
@GetMapping("/all")
@PreAuthorize("hasRole('ADMIN')")
public ResponseEntity<Page<UserProfileDto>> getAllProfiles(
        @PageableDefault(size = 10, sort = "lastName") Pageable pageable) {
    return ResponseEntity.ok(userProfileService.getAll(pageable));
}

@PageableDefault: Setează valorile implicite dacă clientul nu trimite parametri. Altfel, fără ?page=0&size=10, Spring ar returna toate înregistrările.

Service – transmite Pageable la repository

// UserProfileService.java
public Page<UserProfileDto> getAll(Pageable pageable) {
    return userProfileRepository
        .findAll(pageable)          // Spring Data face automat paginarea în SQL
        .map(UserProfileDto::from); // transformă fiecare UserProfile în DTO
}

Repository – moștenit din JpaRepository

// UserProfileRepository.java
public interface UserProfileRepository extends JpaRepository<UserProfile, Long> {
    // findAll(Pageable) este deja disponibil din JpaRepository
    // nu trebuie să scrii nimic suplimentar!
}

// AnnouncementRepository.java – cu query customizat
public interface AnnouncementRepository extends MongoRepository<Announcement, String> {
    Page<Announcement> findByEnabledTrue(Pageable pageable);
    Page<Announcement> findByEnabledTrueAndTagsName(String tagName, Pageable pageable);

Câmpurile clasei Pageable

ParametruTipDescriere
pageintNumărul paginii (0-indexed, prima pagină = 0)
sizeintCâte elemente pe pagină
sortStringcâmp,direcție (ex: lastName,asc)

Configurare globală Pageable (opțional)

// application.yml
spring:
  data:
    web:
      pageable:
        default-page-size: 10
        max-page-size: 100  # previne request de 10000 elemente

Best practice: Setează întotdeauna un max-page-size pentru a preveni atacuri DoS prin request-uri cu size=1000000. Spring va refuza automat request-urile cu size mai mare.

Întrebări frecvente la examen

Î: Ce returnează un endpoint cu Page<T> față de List<T>?
Page<T> este un wrapper care conține: content (lista efectivă de elemente), totalElements (total recorduri în baza de date), totalPages (total pagini), size (elemente per pagină), number (pagina curentă, 0-indexed), first/last (boolean-uri). Un simplu List<T> returnează toate recordurile fără niciun metadata de paginare.
Î: Ce face @PageableDefault și când este necesar?
@PageableDefault(size = 10, sort = "lastName") setează valorile implicite pentru Pageable dacă clientul nu trimite parametrii ?page=&size=&sort=. Fără această adnotare, Spring ar folosi valorile default ale framework-ului (page=0, size=20, fără sortare), ceea ce poate returna mai mult decât e dorit. Este o bună practică să specifici întotdeauna un sort default pentru rezultate deterministe.
Î: Ce SQL generează Hibernate pentru pagina 2 cu size=5?
Hibernate generează SELECT * FROM user_profiles ORDER BY last_name ASC LIMIT 5 OFFSET 5. OFFSET = page × size = 2 × 5 = 10... dar pagina 2 (0-indexed) = a 3-a pagină = OFFSET 10. Dacă însebi pagina 1 (0-indexed) = OFFSET 5. Spring calculează automat OFFSET = pageNumber × pageSize, deci nu trebuie să gestionezi manual paginarea SQL.
Î: Cum trimiți o sortare după mai multe câmpuri?
Prin mai mulți parametri sort în URL: ?sort=lastName,desc&sort=firstName,asc. Spring Data parsează automat fiecare parametru și construiește un Sort compus. Ordinea parametrilor contează: primul sort este prioritar, al doilea e tiebreaker. Alternativ în Java: Sort.by(Sort.Direction.DESC, "lastName").and(Sort.by(Sort.Direction.ASC, "firstName")).
Î: De ce se configurează max-page-size și ce se întâmplă fără el?
Fără max-page-size, un client malițios poate trimite ?size=1000000 și forța serverul să încarce un milion de recorduri din baza de date în memorie — un atac DoS eficient. Cu spring.data.web.pageable.max-page-size: 100, Spring refuză automat valorile mai mari și le setează la maxim. Este o protecție simplă dar importantă.
Î: Cum definești o metodă în repository cu paginare și filtru?
Adaugi parametrul Pageable la orice metodă derivată și returnezi Page<T>. Exemplu: Page<Announcement> findByEnabledTrueAndTagsName(String tagName, Pageable pageable). Spring Data generează automat SQL-ul cu WHERE enabled = true AND tags.name = ? plus LIMIT/OFFSET din Pageable. Nu trebuie să scrii nimic în implementare.

Implementare pas cu pas — Paginare și Sortare

Prerequisite: Entitate JPA existentă cu Repository (Tutorial 02). Spring Data JPA în pom.xml.

Pasul 1 — Activează limitarea paginii în application.yml 
spring:
  data:
    web:
      pageable:
        default-page-size: 10
        max-page-size: 100
Pasul 2 — Actualizează Repository cu Pageable și JPQL 
@Repository
public interface UserRepository extends JpaRepository<User, Long> {

    Page<User> findAll(Pageable pageable);

    @Query("SELECT u FROM User u WHERE " +
           "(:search IS NULL OR LOWER(u.username) LIKE LOWER(CONCAT('%', :search, '%'))) " +
           "AND (:role IS NULL OR u.role = :role)")
    Page<User> findFiltered(@Param("search") String search,
                            @Param("role") Role role,
                            Pageable pageable);
}

JPQL-ul cu :param IS NULL face filtrul opțional — dacă nu se trimite parametrul, condiția e ignorată.

Pasul 3 — Service și Controller

Service:

public Page<UserDto> findFiltered(String search, Role role, Pageable pageable) {
    return userRepository.findFiltered(search, role, pageable).map(UserDto::from);
}

Controller — @PageableDefault setează valorile implicite când clientul nu trimite paginare:

@GetMapping
public ResponseEntity<Page<UserDto>> getUsers(
        @RequestParam(required = false) String search,
        @RequestParam(required = false) Role role,
        @PageableDefault(size = 10, sort = "username", direction = Sort.Direction.ASC) Pageable pageable) {
    return ResponseEntity.ok(userService.findFiltered(search, role, pageable));
}
