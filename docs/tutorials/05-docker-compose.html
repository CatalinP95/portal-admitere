🐳 Docker Compose

Infrastructura completă cu un singur comandă – MySQL, RabbitMQ, Redis, Prometheus, Grafana

Ce este Docker Compose?

Docker Compose definește și rulează mai multe containere Docker simultan folosind un fișier YAML. Ideal pentru medii de dezvoltare: nu trebuie să instalezi MySQL, RabbitMQ etc. local – totul rulează în containere izolate.

Avantaj: Un singur docker compose up -d pornește toate serviciile de infrastructură. Totul rulează izolat, nu "poluează" Windows-ul cu instalări globale.

docker-compose.yml – fișierul complet

version: '3.8'

services:

  mysql:
    image: mysql:8.0
    container_name: portal-mysql
    environment:
      MYSQL_ROOT_PASSWORD: ""
      MYSQL_ALLOW_EMPTY_PASSWORD: "yes"
      MYSQL_DATABASE: portal_admitere_users
    ports:
      - "3306:3306"
    volumes:
      - mysql-data:/var/lib/mysql

  rabbitmq:
    image: rabbitmq:3-management
    container_name: portal-rabbitmq
    ports:
      - "5672:5672"   # AMQP (conexiune aplicație)
      - "15672:15672" # Management UI
    environment:
      RABBITMQ_DEFAULT_USER: guest
      RABBITMQ_DEFAULT_PASS: guest

  redis:
    image: redis:7-alpine
    container_name: portal-redis
    ports:
      - "6379:6379"

  prometheus:
    image: prom/prometheus
    container_name: portal-prometheus
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
    ports:
      - "9090:9090"

  grafana:
    image: grafana/grafana
    container_name: portal-grafana
    ports:
      - "3000:3000"
    environment:
      - GF_SECURITY_ADMIN_PASSWORD=admin
    depends_on:
      - prometheus

volumes:
  mysql-data:

Comenzi Docker Compose

PS portal-admitere> docker compose up -d

Pornește toate containerele în background (-d = detached). Prima rulare descarcă imaginile.

PS portal-admitere> docker compose down

Oprește și șterge containerele (datele MySQL sunt păstrate în volume).

PS portal-admitere> docker compose ps

Listează statusul tuturor containerelor.

PS portal-admitere> docker compose logs rabbitmq -f

Urmărește log-urile unui serviciu specific în timp real (-f = follow).

Cum se leagă Spring Boot de Docker

# application.yml – Spring Boot se conectează la serviciile Docker
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/portal_admitere_users
                           # ↑ containerul MySQL ascultă pe portul 3306 al hostului
  rabbitmq:
    host: localhost
    port: 5672
  data:
    redis:
      host: localhost
      port: 6379

host.docker.internal: În prometheus.yml, target-urile sunt host.docker.internal:8081 (nu localhost), deoarece Prometheus rulează ÎN container. Din containerul Prometheus, "localhost" ar fi containerul însuși, nu Windows-ul. host.docker.internal rezolvă la IP-ul mașinii gazdă.

Volum persistent pentru MySQL

Datele MySQL sunt stocate în volumul Docker mysql-data. Chiar dacă faci docker compose down, datele rămân. Sunt șterse doar cu:

PS portal-admitere> docker compose down -v

Atenție: docker compose down -v șterge volumele și implicit toate datele din MySQL! Folosește doar dacă vrei un fresh start complet.

Întrebări frecvente la examen

Î: Ce face docker compose up -d și ce înseamnă flag-ul -d?
docker compose up pornește toate serviciile definite în docker-compose.yml. Flag-ul -d (detached) le pornește în background — terminalul rămâne liber. Prima rulare descarcă imaginile de pe Docker Hub. Fără -d, log-urile tuturor containerelor sunt afișate în terminal și Ctrl+C oprește totul.
Î: Care este diferența dintre docker compose down și docker compose down -v?
docker compose down — oprește și șterge containerele, dar volumele Docker rămân. Datele din MySQL sunt păstrate. docker compose down -v — oprește containerele ȘI șterge volumele, inclusiv datele MySQL. Folosit pentru un fresh start complet. Atenție: -v este ireversibil!
Î: De ce în prometheus.yml target-ul este host.docker.internal și nu localhost?
Prometheus rulează în container Docker. Din perspectiva containerului, localhost = containerul însuși (nu Windows-ul). host.docker.internal este un hostname special rezolvat la IP-ul mașinii gazdă (Windows). Astfel Prometheus poate ajunge la Spring Boot care rulează direct pe Windows la host.docker.internal:8081.
Î: Ce este un Docker volume și de ce e important pentru MySQL?
Un Docker volume (mysql-data:/var/lib/mysql) este un director gestionat de Docker care persistă datele independent de containerul în care este montat. Dacă containerul MySQL este șters și recreat (ex: upgrade versiune), datele rămân în volum. Fără volum, datele s-ar pierde la fiecare docker compose down.
Î: Ce face depends_on în docker-compose.yml?
depends_on: [prometheus] în configurarea Grafana specifică că Grafana nu pornește înainte de Prometheus. Docker Compose respectă ordinea de pornire. Atenție: depends_on verifică doar că containerul a pornit, nu că serviciul din el este ready (MySQL poate dura câteva secunde să accepte conexiuni după pornire).
Î: Cum verifici că un container rulează corect și cum urmărești log-urile?
docker compose ps — listează statusul (Up/Exit) pentru fiecare serviciu. docker compose logs <serviciu> — afișează log-urile complete. docker compose logs <serviciu> -f — urmărești log-urile în timp real (-f = follow). docker compose logs --tail=50 mysql — ultimele 50 linii de log din MySQL.
Î: Cum se conectează Spring Boot la serviciile din Docker Compose?
Spring Boot rulează în afara Docker (direct pe Windows), deci se conectează la containerele prin porturile expuse pe localhost. În application.yml: MySQL → jdbc:mysql://localhost:3306/..., Redis → host: localhost, port: 6379, RabbitMQ → host: localhost, port: 5672. Dacă Spring Boot ar rula și el în container, s-ar folosi numele serviciului (ex: mysql:3306) prin rețeaua internă Docker.

Implementare pas cu pas — Docker Compose

Prerequisite: Docker Desktop instalat și pornit (iconița din system tray verde). Vezi pagina de prerequisite.

Pasul 1 — Creează docker-compose.yml în rădăcina proiectului

Creează fișierul docker-compose.yml la nivelul portal-admitere/:

version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: portal-mysql
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: portaldb
    ports:
      - "3306:3306"       # portul hostului : portul containerului
    volumes:
      - mysql_data:/var/lib/mysql   # persistență date

  redis:
    image: redis:7-alpine
    container_name: portal-redis
    ports:
      - "6379:6379"

  rabbitmq:
    image: rabbitmq:3-management
    container_name: portal-rabbitmq
    ports:
      - "5672:5672"       # AMQP (aplicații)
      - "15672:15672"     # Management UI (browser)

  prometheus:
    image: prom/prometheus
    container_name: portal-prometheus
    ports:
      - "9090:9090"
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml

  grafana:
    image: grafana/grafana
    container_name: portal-grafana
    ports:
      - "3000:3000"
    volumes:
      - ./grafana/provisioning:/etc/grafana/provisioning
      - ./grafana/dashboards:/var/lib/grafana/dashboards

volumes:
  mysql_data:
Pasul 2 — Creează prometheus.yml

Creează prometheus.yml în rădăcina proiectului:

global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'user-service'
    metrics_path: '/actuator/prometheus'
    static_configs:
      - targets: ['host.docker.internal:8081']
        # host.docker.internal = adresa calculatorului gazdă, accesibilă din container
Pasul 3 — Pornește infrastructura

Din folderul rădăcină al proiectului (unde se află docker-compose.yml):

# Prima oară — descarcă imaginile (3-5 minute)
docker compose up -d

# Verifică că toate containerele rulează
docker compose ps

# Vizualizează log-urile unui serviciu (ex: mysql)
docker compose logs mysql
Pasul 4 — Comenzi utile 
# Oprire fără ștergerea volumelor (datele din MySQL se păstrează)
docker compose stop

# Pornire din nou
docker compose start

# Oprire ȘI ștergere containere (datele din volume se păstrează)
docker compose down

# Oprire ȘI ștergere containere ȘI volume (MySQL se resetează!)
docker compose down -v

# Repornire un singur serviciu (ex: grafana)
docker compose up -d --force-recreate grafana

Verificare: După docker compose up -d, testează că fiecare serviciu răspunde:

MySQL:    docker exec portal-mysql mysql -u root -proot -e "SELECT 1"
Redis:    docker exec portal-redis redis-cli ping          # răspuns: PONG
RabbitMQ: http://localhost:15672  (guest/guest)
Prometheus: http://localhost:9090
Grafana:  http://localhost:3000  (admin/admin)
