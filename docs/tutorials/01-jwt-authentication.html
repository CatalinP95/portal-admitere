🔐 JWT Authentication

JSON Web Tokens în Spring Boot 3 – autentificare stateless completă

Ce este JWT?

JWT (JSON Web Token) este un standard pentru transmiterea securizată a informațiilor între două părți sub forma unui obiect JSON. Un token JWT este semnat digital, deci poate fi verificat fără o bază de date.

Structura unui token JWT:
header.payload.signature
Exemplu: eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiIxIn0.abc123

Flux de autentificare

Client trimite credentials
→
AuthService verifică user
→
JwtUtil generează token
→
Client primește access + refresh token
→
Fiecare request include Authorization: Bearer
→
JwtAuthFilter validează tokenul

JwtUtil – generare și validare

Clasa JwtUtil se află în security/JwtUtil.java și are responsabilitatea de a crea și valida tokeni.

@RefreshScope   // permite reîncărcarea valorilor din Config Server fără restart
@Component
public class JwtUtil {

    @Value("${jwt.secret}")
    private String secret;

    @Value("${jwt.expiration}")
    private long expiration;  // 86400000 = 24 ore în ms

    @Value("${jwt.refresh-expiration}")
    private long refreshExpiration;  // 604800000 = 7 zile în ms

    private SecretKey getKey() {
        return Keys.hmacShaKeyFor(secret.getBytes(StandardCharsets.UTF_8));
    }

    public String generateToken(String userId, String role) {
        return buildToken(userId, role, expiration);
    }

    public String generateRefreshToken(String userId, String role) {
        return buildToken(userId, role, refreshExpiration);
    }

    // permite token cu durată custom (ex: remember me)
    public String generateTokenWithExpiry(String userId, String role, long customExpiryMs) {
        return buildToken(userId, role, customExpiryMs);
    }

    private String buildToken(String subject, String role, long ttl) {
        return Jwts.builder()
                .id(UUID.randomUUID().toString())
                .subject(subject)         // userId ca string
                .claim("role", role)      // ex: "STUDENT", "ADMIN"
                .issuedAt(new Date())
                .expiration(new Date(System.currentTimeMillis() + ttl))
                .signWith(getKey())
                .compact();
    }

    public boolean isTokenValid(String token) {
        try {
            return !extractAllClaims(token).getExpiration().before(new Date());
        } catch (Exception e) {
            return false;
        }
    }
}

Remember Me – token cu durată extinsă

Dacă utilizatorul bifează "Ține-mă conectat", generăm tokeni cu durată mai mare: access 7 zile, refresh 30 zile.

// în LoginRequest.java
private boolean rememberMe = false;

// în AuthService.java
private static final long REMEMBER_ME_ACCESS_MS  = 7L  * 24 * 3600 * 1000;
private static final long REMEMBER_ME_REFRESH_MS = 30L * 24 * 3600 * 1000;

long accessExpiry  = request.isRememberMe() ? REMEMBER_ME_ACCESS_MS  : accessExpirationMs;
long refreshExpiry = request.isRememberMe() ? REMEMBER_ME_REFRESH_MS : refreshExpirationMs;

String accessToken  = jwtUtil.generateTokenWithExpiry(userId, role, accessExpiry);
String refreshToken = jwtUtil.generateTokenWithExpiry(userId, role, refreshExpiry);

JwtAuthFilter – filtrarea requesturilor

Fiecare request HTTP trece prin acest filtru. Dacă există un token valid în header, utilizatorul este autentificat automat.

@Component
public class JwtAuthFilter extends OncePerRequestFilter {

    @Override
    protected void doFilterInternal(HttpServletRequest req,
                                     HttpServletResponse res,
                                     FilterChain chain) {
        String header = req.getHeader("Authorization");

        if (header != null && header.startsWith("Bearer ")) {
            String token = header.substring(7);

            if (jwtUtil.isTokenValid(token)) {
                String userId = jwtUtil.extractUserId(token);
                String role   = jwtUtil.extractRole(token);

                // setăm autentificarea în SecurityContext
                UsernamePasswordAuthenticationToken auth =
                    new UsernamePasswordAuthenticationToken(
                        userId, null,
                        List.of(new SimpleGrantedAuthority("ROLE_" + role))
                    );
                SecurityContextHolder.getContext().setAuthentication(auth);
            }
        }
        chain.doFilter(req, res);
    }
}

Configurare Spring Security

@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    return http
        .csrf(csrf -> csrf.disable())
        .sessionManagement(s -> s.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/api/auth/**").permitAll()
            .requestMatchers("/actuator/**").permitAll()
            .anyRequest().authenticated()
        )
        .addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class)
        .build();
}

De ce stateless?

Serverul nu stochează sesiuni. Fiecare request este independent – util în microservicii unde mai multe instanțe rulează simultan. Tokenul conține toată informația necesară autentificării.

Atenție: Nu stoca informații sensibile în payload-ul JWT (parole, date de card etc.) deoarece poate fi decodat de oricine. Doar semnat, nu criptat!

Întrebări frecvente la examen

Î: Ce conține un token JWT și de ce poate fi decodat de oricine?
Un JWT are 3 părți: header (algoritmul de semnare), payload (claims: sub=userId, role, iat, exp) și signature. Header și payload sunt doar Base64-encoded, nu criptate — oricine le poate decoda. Signature este HMAC-SHA256 cu cheia secretă, deci nu poate fi falsificat fără cheie. Concluzie: nu pune date sensibile în payload!
Î: De ce JWT este „stateless" și care e avantajul în microservicii?
Serverul nu stochează nicio sesiune — tokenul conține toată informația necesară (userId, rol, expiry). Fiecare instanță a serviciului poate valida orice token independent, fără o bază de date de sesiuni partajată. Ideal pentru microservicii unde rulează multiple instanțe simultan.
Î: Care este diferența dintre access token și refresh token?
Access token: durată scurtă (24h implicit, 7 zile cu Remember Me), trimis la fiecare request în header Authorization: Bearer .... Refresh token: durată lungă (7 zile implicit, 30 zile cu Remember Me), folosit doar pentru a obține un nou access token la endpoint-ul POST /api/auth/refresh când access token-ul expiră.
Î: Ce face JwtAuthFilter și cum se integrează în Spring Security?
JwtAuthFilter extends OncePerRequestFilter — rulează o singură dată per request. Extrage tokenul din header-ul Authorization: Bearer ..., validează cu jwtUtil.isTokenValid(), extrage userId și rol, creează un UsernamePasswordAuthenticationToken și îl pune în SecurityContextHolder. Spring Security îl aplică înainte de UsernamePasswordAuthenticationFilter via .addFilterBefore(...).
Î: Ce face @RefreshScope pe JwtUtil și de ce e necesar?
@RefreshScope marchează bean-ul ca „refresh-able": la un POST /actuator/refresh, Spring distruge instanța curentă și creează una nouă cu valorile @Value proaspăt citite din Config Server. Fără această adnotare, JwtUtil ar păstra secreta și expiry-ul de la pornirea aplicației, indiferent de câte ori trimiți refresh.
Î: Cum funcționează Remember Me în implementarea noastră?
LoginRequest are câmpul boolean rememberMe = false. În AuthService.login(), dacă isRememberMe() este true, se apelează generateTokenWithExpiry(userId, role, customMs) cu 7 zile pentru access și 30 zile pentru refresh. Altfel, se folosesc generateToken() și generateRefreshToken() cu valorile standard din Config Server.
Î: Ce cod HTTP se întoarce pentru un request fără token sau cu token expirat?
401 Unauthorized — returnat automat de Spring Security dacă SecurityContext nu conține o autentificare validă. 403 Forbidden — dacă tokenul e valid dar userul nu are rolul necesar (ex: @PreAuthorize("hasRole('ADMIN')") accesat de un STUDENT). Spring Security le gestionează automat prin AuthenticationEntryPoint și AccessDeniedHandler.
Î: De ce excludem MongoRepositoriesAutoConfiguration în testele JwtUtil?
Testele unitare pentru JwtUtil sunt rescrise fără context Spring (fără @SpringBootTest), folosind direct new JwtUtil() și ReflectionTestUtils.setField() pentru câmpurile @Value. Astfel se evită complet problema @RefreshScope (care necesită Spring Cloud) și lipsa mongoTemplate — testele rulează instant, fără nicio dependință externă.

Pasul 2 — Creează JwtUtil

Creează fișierul security/JwtUtil.java:

@Component
@RefreshScope
public class JwtUtil {

    @Value("${jwt.secret}")
    private String secret;

    @Value("${jwt.expiration:86400000}")
    private long expiration;

    @Value("${jwt.refresh-expiration:604800000}")
    private long refreshExpiration;

    private Key getSignKey() {
        byte[] keyBytes = Decoders.BASE64.decode(secret);
        return Keys.hmacShaKeyFor(keyBytes);
    }

    public String generateToken(String userId, String role) {
        return Jwts.builder()
                .setSubject(userId)
                .claim("role", role)
                .setIssuedAt(new Date())
                .setExpiration(new Date(System.currentTimeMillis() + expiration))
                .signWith(getSignKey(), SignatureAlgorithm.HS256)
                .compact();
    }

    public String extractUserId(String token) {
        return Jwts.parserBuilder().setSigningKey(getSignKey()).build()
                .parseClaimsJws(token).getBody().getSubject();
    }

    public boolean isTokenValid(String token) {
        try {
            Jwts.parserBuilder().setSigningKey(getSignKey()).build().parseClaimsJws(token);
            return true;
        } catch (JwtException e) { return false; }
    }
}
Pasul 3 — Creează JwtAuthFilter

Creează security/JwtAuthFilter.java. Acest filtru se execută la fiecare request HTTP:

@Component
public class JwtAuthFilter extends OncePerRequestFilter {

    private final JwtUtil jwtUtil;
    private final UserDetailsService userDetailsService;

    // constructor injection...

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) throws ServletException, IOException {
        // 1. Extrage headerul Authorization
        String authHeader = request.getHeader("Authorization");
        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            filterChain.doFilter(request, response); // lasă requestul să continue
            return;
        }

        // 2. Extrage token-ul (după "Bearer ")
        String token = authHeader.substring(7);

        // 3. Validează și autentifică
        if (jwtUtil.isTokenValid(token)) {
            String userId = jwtUtil.extractUserId(token);
            UserDetails userDetails = userDetailsService.loadUserByUsername(userId);
            UsernamePasswordAuthenticationToken auth =
                new UsernamePasswordAuthenticationToken(userDetails, null, userDetails.getAuthorities());
            auth.setDetails(new WebAuthenticationDetailsSource().buildDetails(request));
            SecurityContextHolder.getContext().setAuthentication(auth); // înregistrează autentificarea
        }

        filterChain.doFilter(request, response); // continuă cu filtrul următor
    }
}
Pasul 4 — Creează endpoint-ul de login

Creează dto/LoginRequest.java:

public record LoginRequest(String username, String password, boolean rememberMe) {}

Creează dto/AuthResponse.java:

public record AuthResponse(String accessToken, String refreshToken) {}

În service/AuthService.java, metoda de login:

public AuthResponse login(LoginRequest request) {
    // 1. Verifică că userul există
    User user = userRepository.findByUsername(request.username())
            .orElseThrow(() -> new BadCredentialsException("Invalid credentials"));

    // 2. Verifică parola cu BCrypt
    if (!passwordEncoder.matches(request.password(), user.getPassword())) {
        throw new BadCredentialsException("Invalid credentials");
    }

    // 3. Generează token-urile
    String accessToken = jwtUtil.generateToken(user.getId().toString(), user.getRole().name());
    String refreshToken = jwtUtil.generateRefreshToken(user.getId().toString());

    // 4. Salvează refresh token în DB (pentru invalidare la logout)
    saveRefreshToken(user, refreshToken, refreshExpirationMs);

    return new AuthResponse(accessToken, refreshToken);
}

În api/AuthController.java:

@RestController
@RequestMapping("/auth")
public class AuthController {

    @PostMapping("/login")
    public ResponseEntity<AuthResponse> login(@RequestBody LoginRequest request) {
        return ResponseEntity.ok(authService.login(request));
    }
}
