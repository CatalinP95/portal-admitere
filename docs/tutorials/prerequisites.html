Timp estimat: 30–45 minute pentru prima instalare completă. Dacă ai deja unele instalate, verifică versiunile — versiuni mai vechi pot cauza erori.

1. Java 21 (Temurin)

De ce Java 21? Spring Boot 3.x necesită minim Java 17. Java 21 este LTS (Long Term Support) și adaugă funcționalități moderne (records, sealed classes, virtual threads).

1

Descarcă Java 21 Temurin

Mergi la adoptium.net → secțiunea "Latest LTS" → selectează Java 21 → Windows x64 MSI Installer.

Sau descarcă direct de la:
https://adoptium.net/temurin/releases/?version=21&os=windows&arch=x64&package=jdk

2

Instalează cu opțiunile implicite

Rulează MSI-ul descărcat. La pasul "Custom Setup" bifează "Set JAVA_HOME variable" și "Add to PATH" dacă sunt disponibile.

3

Verifică instalarea

Deschide un terminal nou (PowerShell sau CMD) și rulează:

java -version

Trebuie să vezi: ✓ openjdk version "21.x.x" ...

echo $env:JAVA_HOME

Trebuie să returneze calea către JDK (ex: C:\Program Files\Eclipse Adoptium\jdk-21...)

Problemă frecventă: Dacă java -version arată o versiune veche, înseamnă că ai mai multe JDK instalate. Setează manual JAVA_HOME: System Properties → Environment Variables → JAVA_HOME → calea JDK 21.

2. Maven 3.9+

De ce Maven? Maven este build tool-ul care compilează codul Java, rulează testele și împachetează aplicația în JAR. Spring Boot folosește Maven (sau Gradle) pentru a gestiona dependențele.

1

Descarcă Maven

Mergi la maven.apache.org/download → descarcă apache-maven-3.9.x-bin.zip

2

Extrage și configurează PATH

Extrage arhiva la o cale simplă, ex: C:\tools\apache-maven-3.9.10\

Adaugă C:\tools\apache-maven-3.9.10\bin la variabila de sistem PATH:

Start → "Environment Variables" → System Variables → Path → Edit → New → calea\bin

3

Verifică instalarea

Deschide un terminal NOU (important — PATH se actualizează doar în terminale noi):

mvn -version

Trebuie să vezi: ✓ Apache Maven 3.9.x ... Java version: 21...

3. Node.js 20+ și npm

De ce Node.js? Angular CLI rulează pe Node.js. npm (Node Package Manager) instalează librăriile Angular (HttpClient, Router, Forms etc.).

1

Descarcă Node.js LTS

Mergi la nodejs.org → descarcă versiunea LTS (minim 20.x) → Windows Installer (.msi)

2

Instalează cu setările implicite

Rulează installerul. Node.js se adaugă automat în PATH.

3

Verifică și instalează Angular CLI

node -v     # trebuie: v20.x.x sau mai nou
npm -v      # trebuie: 10.x.x sau mai nou

npm install -g @angular/cli@19

Verifică Angular CLI:

ng version

Trebuie să vezi: ✓ Angular CLI: 19.x.x

4. Docker Desktop

De ce Docker? Proiectul folosește Docker Compose pentru a porni infrastructura: MySQL, Redis, RabbitMQ, Prometheus, Grafana. Fără Docker ar trebui instalate și configurate manual toate aceste servicii.

1

Descarcă Docker Desktop

Mergi la docker.com/products/docker-desktop → Download for Windows

2

Instalează și activează WSL2

Rulează installerul. Dacă cere activarea WSL2 (Windows Subsystem for Linux), acceptă — Docker Desktop pe Windows îl necesită.

Repornește calculatorul după instalare.

3

Verifică instalarea

Pornește Docker Desktop (iconița din system tray trebuie să fie verde/running).

docker -v
docker compose version

Trebuie să vezi: ✓ Docker version 27.x.x și Docker Compose version v2.x.x

Windows Home: Docker Desktop funcționează pe Windows Home cu WSL2. Asigură-te că virtualizarea este activată în BIOS (Hyper-V sau SVM).

1

Descarcă VS Code

Mergi la code.visualstudio.com → Download for Windows

Apasă Ctrl+Shift+P → scrie "Java: Configure Java Runtime" → verifică că apare JDK 21.

Dacă nu apare: Ctrl+Shift+P → "Java: Configure Java Runtime" → Add JDK → navighează la folderul JDK 21.

6. Git

1

Descarcă și instalează Git

Mergi la git-scm.com/download/win → descarcă installerul 64-bit.

La instalare: la "Choosing the default editor used by Git" alege VS Code. Restul setărilor — implicit.

2

Configurează identitatea

git config --global user.name "Prenume Nume"
git config --global user.email "email@example.com"
3

Verifică instalarea

git --version

Trebuie să vezi: ✓ git version 2.x.x.windows.x

7. Clonarea proiectului

1

Clonează repository-ul

git clone https://github.com/[username]/portal-admitere.git
cd portal-admitere
2

Deschide în VS Code

code .

VS Code va detecta automat proiectele Java (Maven) și Angular.

3

Pornește infrastructura Docker

Din folderul rădăcină al proiectului:

docker compose up -d

Prima dată descarcă imaginile — poate dura 3–5 minute. Verifică că toate containerele rulează:

docker compose ps

Trebuie să vezi toate containerele cu starea Up: mysql, redis, rabbitmq, prometheus, grafana.

Structura proiectului portal-admitere

Proiectul este organizat ca o arhitectură de microservicii:

portal-admitere/
├── config-server/        ← configurare centralizată (Spring Cloud Config)
├── discovery-server/     ← registru servicii (Netflix Eureka)
├── api-gateway/          ← punct de intrare unic (Spring Cloud Gateway)
├── user-service/         ← autentificare, utilizatori, profil (port 8081)
├── admissions-service/   ← admitere, dosare candidați
├── dormitory-service/    ← cămin, locuri, cereri
├── frontend/             ← aplicație Angular (port 4300)
├── docker-compose.yml    ← infrastructura (MySQL, Redis, RabbitMQ, etc.)
└── docs/tutorials/       ← tutorialele tehnice (ești aici)

Ordinea de pornire a serviciilor

  1. docker compose up -d — pornește infrastructura (MySQL, Redis, RabbitMQ, Prometheus, Grafana)
  2. config-server — primul serviciu Spring Boot (portul 8888)
  3. discovery-server — al doilea (portul 8761)
  4. user-service, admissions-service, dormitory-service — se pot porni simultan
  5. api-gateway — ultimul, după ce celelalte s-au înregistrat în Eureka
  6. ng serve — frontend Angular (portul 4300)
